First Game

The purpose of this game was to cut fruit that was placed around the screen, and once the
user cut one fruit, another one would appear. The way the game worked was when the fruit was
touching the wizard, then the fruit would switch costume and then disappear and reappear.

My Game

The purpose of this game was to beat your previous record of the game and keep setting new records.
This game was a combination of "Temple Run" and "Flappy Bird" where in our game the user used
the arrow keys to dodge the moving objects that were heading toward them. Some of the problems that
I faced when I was creating the game was when the flying/moving objects were coming out. I fixed this
problem by broadcasting a call when one of the objects were done in order for the next one to run. With
this kind of program was able to make the game continous and create the game harder as the user
progresses throughout the game, by using or calling different broadcasts.
